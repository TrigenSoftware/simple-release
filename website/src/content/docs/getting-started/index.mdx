**Simple Release** automates the release routine: it reads your git history, determines the next version, updates manifests and changelogs, creates tags and releases, and publishes packages — all in one run. You write commit messages that follow [Conventional Commits](https://www.conventionalcommits.org/), and the tool does the rest, powered by [conventional-changelog](https://conventional-changelog.js.org) under the hood.

- 📄 Parses commits to determine the next version and generate a changelog.
- 🗂️ Supports monorepos — releases multiple packages in a single run, in fixed or independent versioning mode.
- 🧩 Project types, package managers, and hostings are addons — extensible to any stack.
- 🚀 Ships a [GitHub Action](/github-action/) that runs the whole flow in CI: a release pull request, released on merge.

</Aside>

## Not just for JavaScript

Simple Release is written in JavaScript, but it is **platform agnostic by design**. A project is described by an adapter: the [manifest](/js-api/manifest/) knows how to read and write the version file, the [project](/js-api/project/) knows how to bump and publish. The bundled addons cover [npm](/project-types/npm/), [pnpm](/project-types/pnpm/), and [GitHub Actions](/project-types/github-action/) projects, and you can [write your own](/js-api/) for a Python package, a Go module, or anything else with a version and a changelog.

With the JSON [config](/getting-started/configuration/) and the GitHub Action, a non-JavaScript project doesn't even need a `package.json` — the action installs the addons named in the config by itself.
